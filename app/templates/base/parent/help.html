Help

Getting Started

  1. Add content — Go to Add and paste a YouTube (or other) video URL. You can add individual videos, entire channels, or playlists.
  2. Organize into channels — Go to Channels to create categories (e.g. "Science", "Music"). Assign videos to channels when adding or editing them.
  3. Create child profiles — Go to Profiles to create profiles for each child. Set a display name, optional PIN, theme, search permissions, and which channels they can see.
  4. Share the kid URL — Give your kids http://{{ request.headers.get('host', 'localhost:8080') }}/ — they'll see the profile picker and can only watch approved content.

Key Concepts

Channels

Channels are categories you create to organize videos (e.g. "Cartoons", "Science", "Music"). They control what each child profile can see. If a profile has no channels assigned, they see everything. If you assign specific channels, they only see videos in those channels.

Profiles

Each child gets a profile with their own name, avatar, theme, and permissions. Profiles control:

  • Display name — what the child sees in the header
  • PIN — optional, prevents siblings from using each other's profiles
  • Theme — visual style (Default, Playful, Calm)
  • Search — disabled (default) or curated (search within approved content only)
  • Channels — which channels this child can see

Video Downloads

Videos are downloaded when you add them, not when a child watches. The Content tab shows download status. Videos only appear in the kid UI after they're fully downloaded.

Download resolution and cache duration can be configured in Settings. Set cache to 0 days to keep videos forever.

Comments & Reactions

Children can react to videos with emojis and post comments. Comments are only visible between profiles that share channels — a toddler won't see a teen's comments if they don't share any channels. You can delete any comment from the Stats page.

Anti-Bot

If adding videos fails with "bot detection" errors (most often when pasting YouTube URLs), go to Settings and configure browser impersonation (recommended) or cookie authentication. Browser impersonation works without any login. Other yt-dlp-supported sources generally don't need anti-bot configuration.

Troubleshooting

Video won't download

  • Check the error message in the Content tab
  • If it says "bot detection" — enable browser impersonation in Settings
  • If it says "unavailable" — the video may be private or region-locked
  • Check your internet connection

Kid can't see a video

  • The video must be fully downloaded (status: "ready")
  • The video must be in a channel assigned to the child's profile
  • The video status must be "active" (not hidden)

Server won't start

  • Run python run.py and check the error messages
  • Make sure Python 3.10+, ffmpeg, and Deno are installed
  • Run pip install -r requirements.txt

Data & Privacy

  • All data stays on your machine — nothing is sent to any cloud service
  • Kids never connect to the original video source — all video streaming goes through your server
  • Data is stored in ~/curatables-data/ by default (configurable in Settings)
  • Database: curatables.db (SQLite). Videos: videos/ directory
